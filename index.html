    CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want arpund the content inside the element.
  • Visually, from outside to inside: Mardgin -> Border -> Padding -> Content.
  • Note that the declaration Margin: 40px auto; means that the margin top and botton is set to 40 px and automatic on the sides.
  • Box shadow properties are declared in X and Y offsets. X= offset to the right of the box and Y is offset below the box. If not careful, overlaps with other elemesnts can be caused.
  • To add colors, you can either use semantic names of colors or hexadecimal #. Use HTML Color code Website for more information.
  • Declarations can be made using an element name e.g. h1, h2, header, footer etc.
  • But there may be a time when you only need to add css rules to certain elements and not all, in this case you use class attributes in your css declarations e.g. .card.

    Git

  • You need to learn and master Git Flow
  • Make sure your in the main branch. If not use the command: git checkout main
  • Pull the latest content from the main branch using: git pull origin main
  • Create a new branch called feature/add-html: git checkout -b feature/add-html
  • Check the branch in which you are in with: git satus
  • Stage files from the lacal repo to your github repo: git add -A
  • To commit the changes you made, use: git commit -m "added code to HTML file". Note that -m allows us to associate a message to our commit.
  • Push all local changes to main github branch using: git push origin feature/add-html
